# Author Declaration Reply Intake Runbook

Generated by `python scripts/build_author_declaration_email_packet.py`.

## Purpose

Use this runbook after authors reply to the declaration confirmation email or WeChat message. Do not mark any item as confirmed without written evidence.

## Evidence Folder

`metadata/author_reply_evidence/`

## Intake Steps

1. Save each author reply as a `.txt`, `.eml`, screenshot, or exported message under `metadata/author_reply_evidence/`.
2. Name evidence files by item and author, for example `funding_statement_confirmation_yi_miao.txt`.
3. Update `metadata/author_metadata.tsv` only for items explicitly confirmed or corrected by authors.
4. Update `metadata/credit_roles.tsv` only after role approval or correction.
5. For Figure 4 acknowledgement, run `python scripts/record_corresponding_author_signoff.py` after saving the evidence.
6. Re-run `python scripts/build_author_declaration_confirmation_packet.py`.
7. Re-run `python scripts/build_v0_1_1_release_preflight.py` and `python scripts/build_high_impact_submission_dashboard.py`.

## Stop Rules

- Stop if any author changes the author order, affiliation, ORCID, funding, or competing-interest statement.
- Stop if Figure 4 is requested to become a mechanism, clinical, prognosis, therapy-response, or patient-level claim.
- Stop if any private clinical data or identifiable metadata are introduced.
- Stop if the official journal system requests information not represented in the local packet.
