# Statistical Analysis Plan

## Null Model

After library-size normalization, variance stabilization, optional batch
residualization, and whitening, the non-biological noise component is modeled as
a high-dimensional random matrix whose covariance spectrum follows a
Marchenko-Pastur bulk. PCs exceeding the selected finite-sample edge are treated
as candidate signal PCs.

## PC Calibration

RMTGuard v2 supports four PC rules:

- `mp`: Marchenko-Pastur upper edge.
- `mp_tw`: maximum of MP edge and a Johnstone/Tracy-Widom proxy edge.
- `permutation`: gene-wise permutation max-eigenvalue calibration when
  `n_permutations > 0`, otherwise fallback to `mp_tw`.
- `mp_tw_permutation`: conservative maximum of all available edges.

Permutation calibration is recommended for manuscript-grade main figures.

## HVG and Clustering Rules

HVG selection uses the smallest grid value reaching the prespecified fraction of
the maximum observed RMT-supported signal-PC count. Neighborhood size and
cluster granularity are selected by stability after adding RMT-null noise PCs.

## Multiple Testing and Comparisons

- Synthetic null simulations estimate false signal-PC discovery.
- Dataset-level benchmark metrics are paired by method.
- Benjamini-Hochberg correction is applied within metric families.
- Confidence intervals should be generated by bootstrap across samples or
  datasets, not by treating individual cells as independent biological
  replicates.

## Reporting

Each result must report cells, genes, selected HVGs, signal PCs, MP edge,
Tracy-Widom proxy edge, permutation edge when used, MP bulk KS, null
false-positive rate, runtime, peak memory, random seed, and software version.
